# HQ Telecom Support — Docusaurus Site

This site was generated from a Freshdesk Solutions export (`Solutions.xml`)
and contains the full HQ Telecom knowledge base: **14 product categories**
and **128 articles**, converted from HTML into clean Markdown.

## Structure

```
docs/
  intro.md                 # Homepage (served at site root, "/")
  <product-folder>/
    _category_.json        # Sidebar label + auto-generated index page
    <article-slug>.md      # One file per knowledge-base article
```

The sidebar (`sidebars.js`) is fully **autogenerated** from the `docs/`
folder structure, ordered using the `position` values from the original
Freshdesk export (folder order via `_category_.json`, article order via
each file's `sidebar_position` front matter).

Internal links between articles (e.g. "see this related article") were
automatically rewritten to relative Docusaurus links. Images are still
referenced from their original Freshdesk/S3 CDN URLs (this environment
had no network access to download them — see **Optional: localize images**
below if you'd like to self-host them).

Every article has `format: md` in its front matter so Docusaurus parses
it as plain CommonMark rather than MDX — this avoids build errors from
stray `<`, `>`, `{`, `}` characters that are common in HTML-derived content.

## Getting started

Requires [Node.js](https://nodejs.org/) 18+.

```bash
npm install
npm start
```

This starts a local dev server at `http://localhost:3000` with hot reload.

## Build for production

```bash
npm run build
```

Static files are output to `build/`. Serve them with:

```bash
npm run serve
```

## Deploying

The generated `build/` folder is a static site and can be deployed to
GitHub Pages, Netlify, Vercel, Cloudflare Pages, S3, or any static host.
Update `url`, `baseUrl`, `organizationName`, and `projectName` in
`docusaurus.config.js` to match your deployment target before building.

## Optional: localize images

Article images currently point to:
`https://s3.amazonaws.com/cdn.freshdesk.com/...`

If you'd like to host them yourself, download each image into
`static/img/articles/`, then bulk find-and-replace the S3 URLs in
`docs/**/*.md` with `/img/articles/<filename>`.

## Regenerating from a new export

The conversion script used to build this site is included at
`generate.py` (requires `lxml` and `markdownify`: `pip install lxml
markdownify`). Place a new `Solutions.xml` export at the path referenced
in the script, delete the `docs/` folder, and re-run:

```bash
python3 generate.py
```
